Fiche E4 — Stage : Les Réparateurs Mac et PC

Support et mise à disposition de services Rapport de stage

Contexte

Stage réalisé en entreprise spécialisée dans la réparation de matériels informatiques (PC et Mac). Dans le cadre de l'épreuve E4, j'ai assuré des missions de support technique N1/N2 auprès des clients et participé à la gestion du parc informatique.

Environnement technique

  • Systèmes : Windows 10/11, macOS
  • Matériels : PC portables et fixes, Mac, périphériques
  • Outils : Outils de diagnostic matériel/logiciel
  • Bureautique : Suite Microsoft Office, logiciels métier

Compétences mobilisées

Code Compétence Mise en œuvre
B1.1 Gestion du patrimoine informatique Inventaire des matériels, suivi des interventions, gestion des pièces de remplacement
B1.2 Réponse aux incidents et aux demandes d'assistance Diagnostic et résolution d'incidents logiciels et matériels (PC, Mac), prise en charge des clients
B1.4 Travail en mode projet Gestion des priorités d'intervention, suivi des dossiers clients, respect des délais
B1.5 Mise à disposition des utilisateurs d'un service informatique Installation et configuration de postes de travail, remise en état et livraison aux clients
B1.6 Organisation de son développement professionnel Découverte du milieu professionnel, adaptation aux procédures internes, relation client

Difficultés rencontrées et solutions

  • Diversité des pannes : Chaque intervention est différente — développement d'une méthode de diagnostic systématique (logiciel → matériel → système)

  • Gestion du temps : Plusieurs interventions simultanées — priorisation selon l'urgence et le type de panne

Fiche n°1 — Gust'Délice

Développement applicatif

Contexte

Projet personnel réalisé en autonomie dans le cadre du BTS SIO SLAM. L'objectif était de concevoir et développer une application web complète de partage de recettes de desserts, de la base de données jusqu'à l'interface utilisateur.

Environnement technique

  • Serveur : Apache (XAMPP)
  • Langage backend : PHP 8
  • Base de données : MySQL / PhpMyAdmin
  • Frontend : HTML5, CSS3, Bootstrap 5
  • Versioning : Git / GitHub
  • IDE : Visual Studio Code

Compétences mobilisées

Code Compétence Mise en œuvre
B2.1 Conception et développement d'une solution applicative Conception MCD, développement PHP complet (CRUD, sessions, upload)
B2.2 Maintenance corrective ou évolutive Corrections de bugs, ajout du système de commentaires en cours de projet
B2.3 Gestion des données Modélisation BDD MySQL, requêtes PDO préparées, sécurisation des données
B2.4 Travail en mode projet Planification des étapes, suivi sur GitHub
B2.5 Production de documentation Documentation technique, dictionnaire de données, MCD

Difficultés rencontrées et solutions

  • Upload d'images : Gestion des types MIME et des noms de fichiers uniques — résolu avec uniqid() et vérification $_FILES
  • Sécurité des formulaires : Risques d'injections SQL — résolu avec PDO et requêtes préparées
  • Gestion des sessions : Déconnexion automatique et contrôle d'accès aux pages protégées

Fiche n°2 — Les Bonnes Pièces

Développement applicatif

Contexte

Projet personnel simulant un site e-commerce de vente de pièces automobiles. L'objectif était de maîtriser la gestion d'un catalogue produits, d'un système de panier et d'une interface d'administration, dans un contexte proche d'un projet professionnel réel.

Environnement technique

  • Serveur : Apache (XAMPP)
  • Langage backend : PHP 8
  • Base de données : MySQL / PhpMyAdmin
  • Frontend : HTML5, CSS3, Bootstrap 5, JavaScript
  • Versioning : Git / GitHub
  • IDE : Visual Studio Code

Compétences mobilisées

Code Compétence Mise en œuvre
B2.1 Conception et développement d'une solution applicative Architecture MVC simplifiée, panier en session, interface admin complète
B2.2 Maintenance corrective ou évolutive Ajout de fonctionnalités (filtres produits, gestion des stocks)
B2.3 Gestion des données BDD relationnelle (produits, catégories, utilisateurs, commandes), requêtes PDO
B2.4 Travail en mode projet Découpage en sprints, versioning GitHub, cahier des charges
B2.5 Production de documentation Manuel utilisateur, documentation de la base de données, MLD

Difficultés rencontrées et solutions

  • Panier multi-produits : Gestion des quantités et persistance en session PHP — résolu avec un tableau associatif en $_SESSION
  • Interface d'administration : Séparation des droits entre utilisateur simple et admin — résolu avec un champ role en BDD et vérification à chaque page
  • Performances des requêtes : Optimisation des jointures SQL pour le catalogue avec filtres
